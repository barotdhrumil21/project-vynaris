# Plan work

## Step 1 — clarify the outcome
What does "done" look like? One sentence. Write to `plan.md`. If unclear, ask.

## Step 2 — break it down
Produce 3–7 tasks. Each task has:
- A verb + concrete object ("Pull last 90 days of defaults from the data warehouse")
- An expected duration (half-day / 1 day / 2 days — don't use hours for knowledge work)
- A first-step trigger ("Check if we have prod read access by EOD")

## Step 3 — sequence
Which tasks unblock others? Put those first. Mark parallelizable tasks explicitly.

## Step 4 — risks
What are the 2–3 ways this plan could fail? Name them.

## Step 5 — commit
Append to `todo.md` as a checklist. Post to feed: "Plan for <goal>: <one-line summary>. Starting on <first task>."

## Anti-patterns
- Don't produce plans with more than 7 top-level tasks. That's not a plan, that's a list.
- Don't estimate in hours — it's false precision for knowledge work.
- Don't plan past what's visible. After a first chunk, you'll re-plan; say so.
